Selamat datang di dokumentasi E-Complain Frontend! Dokumen ini dirancang khusus untuk pemula yang ingin memahami cara kerja aplikasi Vue.js dari awal.

Apa itu E-Complain?

E-Complain adalah aplikasi web untuk mengelola keluhan/pengaduan. Aplikasi ini memungkinkan:

  • User membuat dan melacak keluhan mereka
  • Admin mengelola dan merespons keluhan
  • Tracking status keluhan secara real-time

Teknologi yang Digunakan

  • Vue.js 3: Framework JavaScript untuk membangun antarmuka pengguna
  • Vue Router: Untuk navigasi antar halaman
  • Pinia: Untuk manajemen state (data) aplikasi
  • TailwindCSS: Untuk styling komponen
  • Axios: Untuk komunikasi dengan backend API

1. Memahami Struktur Proyek

Struktur Folder Dasar

Mari kita pahami struktur folder seperti kita mengorganisir rumah:

  • src/: Ruang utama aplikasi (seperti ruang tamu)
  • components/: Komponen yang bisa digunakan berulang kali (seperti perabotan)
  • views/: Halaman-halaman utama (seperti ruangan)
  • stores/: Tempat menyimpan data (seperti lemari)
  • router/: Pengatur navigasi (seperti peta rumah)

2. Memahami Komponen Vue.js

Apa itu Komponen?

Komponen adalah seperti blok LEGO yang membentuk aplikasi. Setiap komponen memiliki 3 bagian utama:

  • Template: HTML yang ditampilkan (seperti struktur LEGO)
  • Script: Logika JavaScript (seperti instruksi perakitan)
  • Style: CSS untuk tampilan (seperti warna dan dekorasi)

Contoh Komponen Login

<template>
  <div class="login-container">
    <form @submit.prevent="handleLogin">
      <input v-model="email" type="email" />
      <input v-model="password" type="password" />
      <button type="submit">Login</button>
    </form>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useAuthStore } from '@/stores/auth'

// State lokal
const email = ref('')
const password = ref('')

// Store untuk autentikasi
const authStore = useAuthStore()

// Fungsi untuk handle login
const handleLogin = async () => {
  try {
    await authStore.login({
      email: email.value,
      password: password.value
    })
  } catch (error) {
    console.error('Login gagal:', error)
  }
}
</script>

<style scoped>
.login-container {
  padding: 20px;
  max-width: 400px;
  margin: 0 auto;
}
</style>

Konsep Penting dalam Komponen

  • v-model: Mengikat input dengan data (two-way binding)
  • @submit.prevent: Mencegah form melakukan refresh halaman
  • ref: Membuat variabel reaktif
  • scoped: CSS hanya berlaku untuk komponen ini

3. State Management dengan Pinia

Apa itu State Management?

State management adalah seperti buku catatan pusat yang menyimpan semua data penting aplikasi. Pinia membantu kita mengelola data ini dengan terstruktur.

Contoh Store Auth

// stores/auth.js
import { defineStore } from 'pinia'
import { axiosInstance } from '@/plugins/axios'
import Cookies from 'js-cookie'

export const useAuthStore = defineStore('auth', {
  // State: Data yang disimpan
  state: () => ({
    user: null,        // Data user yang login
    loading: false,    // Status loading
    error: null,       // Pesan error
    success: null      // Pesan sukses
  }),

  // Getters: Cara mengambil data
  getters: {
    isAuthenticated: (state) => !!state.user,
    isAdmin: (state) => state.user?.role === 'admin'
  },

  // Actions: Fungsi untuk mengubah data
  actions: {
    async login(credentials) {
      this.loading = true
      try {
        const response = await axiosInstance.post('/login', credentials)
        const token = response.data.data.token
        
        // Simpan token
        Cookies.set('token', token)
        
        // Update state
        this.user = response.data.data.user
        this.success = response.data.message
      } catch (error) {
        this.error = handleError(error)
      } finally {
        this.loading = false
      }
    }
  }
})

Cara Menggunakan Store dalam Komponen

// Dalam komponen
import { useAuthStore } from '@/stores/auth'

const authStore = useAuthStore()

// Menggunakan state
console.log(authStore.user)

// Menggunakan getter
console.log(authStore.isAuthenticated)

// Memanggil action
await authStore.login({
  email: 'user@example.com',
  password: 'password123'
})

4. Routing dengan Vue Router

Apa itu Routing?

Routing adalah seperti peta yang mengatur navigasi antar halaman dalam aplikasi. Vue Router membantu kita mengelola perpindahan halaman dengan mudah.

Konfigurasi Router

// router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/login',
      name: 'login',
      component: () => import('@/views/auth/Login.vue')
    },
    {
      path: '/dashboard',
      name: 'dashboard',
      component: () => import('@/views/app/Dashboard.vue'),
      meta: {
        requiresAuth: true
      }
    }
  ]
})

// Navigation Guard
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  if (to.meta.requiresAuth) {
    if (authStore.token) {
      try {
        await authStore.checkAuth()
        next()
      } catch (error) {
        next('/login')
      }
    } else {
      next('/login')
    }
  } else {
    next()
  }
})

Cara Menggunakan Router dalam Komponen

<template>
  <div>
    <!-- Link untuk navigasi -->
    <router-link to="/dashboard">
      Dashboard
    </router-link>

    <!-- Tombol untuk navigasi programatik -->
    <button @click="goToLogin">
      Login
    </button>
  </div>
</template>

<script setup>
import { useRouter } from 'vue-router'

const router = useRouter()

const goToLogin = () => {
  router.push('/login')
}
</script>

5. Interaksi dengan API menggunakan Axios

Apa itu Axios?

Axios adalah seperti kurir yang mengirim dan menerima data dari server. Kita menggunakannya untuk berkomunikasi dengan backend API.

Konfigurasi Axios

// plugins/axios.js
import axios from 'axios'
import Cookies from 'js-cookie'

export const axiosInstance = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  headers: {
    'Content-Type': 'application/json'
  }
})

// Menambahkan token ke setiap request
axiosInstance.interceptors.request.use(config => {
  const token = Cookies.get('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// Menangani response
axiosInstance.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      Cookies.remove('token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

Cara Menggunakan Axios dalam Komponen

// Dalam komponen atau store
import { axiosInstance } from '@/plugins/axios'

// Mengambil data
const getData = async () => {
  try {
    const response = await axiosInstance.get('/api/data')
    return response.data
  } catch (error) {
    console.error('Error:', error)
  }
}

// Mengirim data
const postData = async (data) => {
  try {
    const response = await axiosInstance.post('/api/data', data)
    return response.data
  } catch (error) {
    console.error('Error:', error)
  }
}

9. Penjelasan Detail untuk Pemula

9.1 Memahami Vue.js dari Awal

A. Apa itu Vue.js dan Bagaimana Cara Kerjanya?

Vue.js adalah framework JavaScript yang membantu kita membuat aplikasi web interaktif. Mari kita pahami dengan analogi sederhana:

1. Komponen Vue.js

Bayangkan komponen Vue.js seperti blok LEGO:

  • Setiap blok (komponen) memiliki fungsi spesifik
  • Blok-blok bisa disusun menjadi struktur yang lebih besar
  • Blok yang sama bisa digunakan di tempat berbeda

2. Data Binding

Data binding adalah cara Vue.js menghubungkan data dengan tampilan. Seperti resep masakan:

  • Data adalah bahan-bahan masakan
  • Tampilan adalah hasil masakan
  • Ketika bahan berubah, hasil masakan otomatis berubah

3. Reaktivitas

Reaktivitas adalah kemampuan Vue.js untuk mendeteksi perubahan data. Seperti termometer:

  • Termometer mendeteksi perubahan suhu
  • Vue.js mendeteksi perubahan data
  • Ketika data berubah, tampilan otomatis diperbarui

B. Struktur Dasar Komponen Vue.js

Setiap komponen Vue.js memiliki 3 bagian utama:

1. Template (HTML)

Template adalah struktur HTML yang akan ditampilkan. Seperti kerangka rumah:

  • Menentukan struktur tampilan
  • Menggunakan direktif Vue (v-if, v-for, dll)
  • Menampilkan data dari script

2. Script (JavaScript)

Script berisi logika komponen. Seperti otak yang mengatur fungsi rumah:

  • Menyimpan dan mengelola data
  • Menangani interaksi pengguna
  • Berkomunikasi dengan komponen lain

3. Style (CSS)

Style menentukan tampilan komponen. Seperti dekorasi rumah:

  • Mengatur warna, ukuran, dan posisi
  • Bisa spesifik untuk komponen ini saja
  • Bisa menggunakan class dari TailwindCSS

9.2 Memahami State Management dengan Pinia

A. Apa itu State Management?

State management adalah cara mengelola data dalam aplikasi. Bayangkan seperti toko:

1. State (Data)

State adalah data yang disimpan. Seperti inventori toko:

  • Data user yang login (seperti kartu member)
  • Data keluhan (seperti daftar pesanan)
  • Status loading (seperti tanda "sedang melayani")

2. Getters (Cara Mengambil Data)

Getters adalah cara untuk mengambil data. Seperti petugas toko:

  • Mencari data berdasarkan kriteria
  • Menghitung atau memproses data
  • Mengembalikan data yang sudah difilter

3. Actions (Cara Mengubah Data)

Actions adalah fungsi untuk mengubah data. Seperti kasir toko:

  • Menambah data baru
  • Mengubah data yang ada
  • Menghapus data

B. Kapan Menggunakan State Management?

State management digunakan ketika:

  • Data digunakan di banyak komponen
  • Data perlu dipertahankan saat berpindah halaman
  • Data perlu diakses secara global

C. Best Practices State Management

Beberapa praktik terbaik dalam menggunakan state management:

  • Pisahkan state berdasarkan fitur
  • Gunakan actions untuk operasi asinkron
  • Gunakan getters untuk data yang perlu diproses
  • Jaga state tetap sederhana dan terorganisir

9.3 Memahami Routing dengan Vue Router

A. Apa itu Routing?

Routing adalah cara mengatur navigasi dalam aplikasi. Bayangkan seperti peta mall:

1. Route (Jalan)

Route adalah path yang menghubungkan halaman. Seperti jalan di mall:

  • Setiap toko memiliki alamat unik
  • Jalan menghubungkan antar toko
  • Petunjuk arah membantu pengunjung

2. Navigation Guards (Petugas Keamanan)

Navigation guards adalah pengecekan sebelum mengizinkan navigasi:

  • Mengecek apakah user sudah login
  • Mengecek apakah user memiliki akses
  • Mengarahkan ke halaman yang sesuai

B. Kapan Menggunakan Routing?

Routing digunakan ketika:

  • Aplikasi memiliki banyak halaman
  • Perlu navigasi antar halaman
  • Perlu mengelola URL aplikasi

C. Best Practices Routing

Beberapa praktik terbaik dalam menggunakan routing:

  • Gunakan named routes untuk kemudahan maintenance
  • Implementasikan lazy loading untuk performa
  • Gunakan meta fields untuk informasi tambahan
  • Tangani error routing dengan baik
